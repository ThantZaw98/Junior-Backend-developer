Q7. Facade Pattern ကို အသုံးပြုတဲ့အတွက် ဘာအကျိုးကျေးဇူးတွေရနိုသ်ပါသလဲ?
facade pattern သုံးခြင်းအားဖြင့် complex ဖြစ်တဲ့ system တစ်ခုကို  facade class မှာ subsystem အဖြစ်ပေါင်းစပ်ပီး တပြိုင်နက်ထဲ   client အလွယ်တကူ ခေါ်ယူသုံး နိုင်အောင် လုပ်ဆောင်ပေးနိုင်တဲ့ အတွက်   အသုံးအများဆုံးdesign pattern တစ်ခုဖြစ်ပါတယ်။ laravel မှာ ကိုယ်ပိုင်  service တစ်ခုထည့် သွင်း ချင်တယ် ဆိုပါစို့ service container တစ်ခုတည်ဆောက်ပြီး laravel ရဲ့ facade pattern ကို မိမိ class တစ်ခုမှာ extends လုပ်ပီး  Implicit Binding အဖြစ် မိမိ service ကို နေရာတိုင်း မှာ ခေါ်ယူသုံးလို့ရမှာ ဖြစ်ပါတယ်။
view() တို့ request() တို့ဆိုတာလည်း facade Pattern တွေထဲက method တွေဖြစ်ပါတယ်။
View::make() အစား view() အဖြစ် client အလွယ်တစ်ခု သုံးစွဲနိုင်ပါဖြစ်ပါတယ်။
